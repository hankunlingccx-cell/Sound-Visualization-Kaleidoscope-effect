0. 执行摘要（与代码一致）

**当前状态：**录音、音频分析、状态机、界面与 WebGL2 **线性万花筒**渲染均已实现（V1.4）。主体由连续曲线骨架 + 平行线束（ribbon mesh）构成，辅以少量沿线微粒；已从 V1.3 点云轮廓升级。

核心约束：

同一 MediaStream 同时用于可视化分析与本地录音；不并行打开第二路麦克风。

曲线顶点和少量粒子的位置在 GPU 顶点着色器中计算；CPU 每帧只传时间与少量音频特征。

先在一个基础半扇区中生成连续曲线，再通过“扇区旋转复制 + 扇区内镜像折叠”形成完整万花筒；所有副本共用 seed、形变、亮度与音频响应。

允许非常缓慢的整体相位漂移与层间反向流动，但禁止快速转盘式旋转、每个扇区独立随机、单侧爆发或破坏整体平衡的方向性拖尾。

麦克风采集需 AudioContext.resume()、约束回退、AGC 默认开启；失败给出可恢复错误，不崩溃。

底栏（.dock）必须保证主按钮始终可见；状态区可独立滚动。

1. 项目背景与范围

1.1 目标体验

用户进入页面后，中央出现由大量细线组成的多层径向结构。它不是一团独立颗粒，也不是单一圆环，而是同时包含中心核、折叠内环、中层回环和外侧放射线束。每一组结构由多条间距细密的平行轮廓线构成，线条可带轻微点状纹理，但整体轮廓必须连续。即使没有声音，各层也会以不同速度缓慢流动、收缩、展开和相互穿插；声音到来后，结构继续连续演化，而不是突然切换图案。

目标观感：

类似实时生成的数字万花筒 / 音频曼陀罗，但避免形成明确花瓣或静态徽章。

任一时刻具有可识别的 6–12 重径向秩序；局部轨迹可弯曲、交叠、伸出。

中心细密明亮，向外逐渐稀疏；外圈不是完整闭合圆环，而是有节奏地长出连续触须、弧带或网状线束。

主体视觉中 85%–95% 的亮度来自连续曲线，游离粒子只占 5%–15%，不能再以粒子云承担主体轮廓。

变化应连续、丝滑、不可预测但不混乱；静音时也持续低速演化。

视觉结构保持整体平衡，但不再要求只关于水平轴和垂直轴形成四份完全相同的复制。

万花筒对称关系：

$$P_{k,\pm}(r,\theta,t)=P(r,\ k\cdot\frac{2\pi}{K}\pm\theta+\phi(t),\ t)$$

其中 K 为旋转扇区数，建议常态取 6 或 8；±θ 产生扇区内镜像，因此最终可形成 12 或 16 个细分方向。

3. 视觉设计规范

3.1 基础外观

背景：#050309

主色：洋红 #FF3DCE、紫色 #7B61FF、浅紫 #C7B7FF；中心允许白粉色高亮

形态：抽象多层、中心偏亮、外缘轻盈；主体必须表现为细密连续线束；禁止粒子堆成块、只出现一个粗椭圆环，或长期停留成固定花瓣/植物轮廓

对称：采用径向万花筒秩序，不再限制为水平/垂直双轴镜像；拖尾与 Bloom 必须保持环向均衡

动态：各层持续呼吸、折叠、穿插、伸缩；允许慢速相位漂移与层间反向运动

主线宽度：0.35–0.9 px；强调线宽度不超过 1.2 px。移动端按 devicePixelRatio 换算，不能因高 DPR 变粗

辅助粒子尺寸：0.45–0.9 px，仅沿曲线切线方向流动，不允许大面积自由散布

混合：SRC_ALPHA, ONE 加法混合；关闭深度测试

色彩：默认保留紫粉品牌色，但整体饱和度降低；高亮粒子趋近银白/浅紫，避免整团均匀紫色。颜色参数应可配置，以便切换为参考图的黑底灰白版本

3.2 曲线、线束与粒子的组成比例（V1.4）

为消除当前“一块一块”的颗粒感，采用：

seed 描述基础半扇区中的曲线骨架，复制数量由 K 决定

每条骨架沿参数 u = 0..1 连续采样，相邻采样点必须连接；禁止把采样点作为互不相连的圆形点精灵直接绘制

每条骨架沿法线方向偏移生成 8–28 条平行细线，形成参考图中的密集轮廓线束

每条线建议 96–240 个纵向采样点；近中心可更密，外层可稍稀

相邻线的间距建议为 1.2–3.5 px，并随半径轻微扩大；不得随机聚成斑块

使用 4 个结构层，不把全部曲线放在同一半径；每层具有独立的基础半径、曲率、相位和流速

同一 seed 的复制体共享噪声，避免扇区之间抖动不一致

形变只使用低频连续噪声、正弦组合或阻尼插值，禁止逐帧重新随机

仅保留少量微粒沿曲线切线方向流动，作为“声音能量经过线束”的提示；不得脱离骨架形成大面积星尘

声音只能调制连续运动的幅度与速度，不能完全决定形态。推荐：

finalParam = autonomousBase(time, seed) + audioFeature * audioWeight;
// autonomousBase 权重约 60%–75%，audioWeight 约 25%–40%

这样静音时仍然不断变换，声音只让万花筒“更有生命”，不会在无声时停住。

平滑公式：

smoothed = previous + (target - previous) * factor
factor = target > previous ? 0.35 : 0.08   // 视觉参数默认
silentBreath = 0.5 + 0.5 * sin(time * 0.8)
effectiveVolume = max(smoothedVolume, silentBreath * 0.04)

静音时必须保留最低呼吸，禁止全黑死寂。

4. 万花筒曲线形态与 Shader 规则

4.1 基础半扇区的参数曲线模型

每条基础曲线只在半扇区 θ ∈ [0, π/K] 中生成一次。每个顶点保存：

curveId：曲线骨架编号

u：沿曲线方向的位置，范围 0..1；同一条线上必须按顺序连续

strandId：同一骨架内的平行线编号

strandOffset：相对骨架的法向偏移

layer：Core / Inner / Mid / Outer

random1..2：曲线级稳定随机值，不允许每个顶点各自乱抖

phase：该线束的初始相位

先根据 u 计算一条连续中心线 C(u)，再用中心线导数获得法线，将中心线偏移为一组平行线：

vec2 centerline(float u, float curveId, float layer) {
    float sectorAngle = TAU / float(uSectorCount);
    float theta = u * sectorAngle * 0.5;

    // 低频连续形变；相邻 u 的输入连续，不能混入逐顶点白噪声
    float waveA = sin(u * TAU * uWaveOrderA + uTime * 0.23 + curveId);
    float waveB = cos(u * TAU * uWaveOrderB - uTime * 0.17 + layer);
    float r = layerBaseRadius(layer, curveId);
    r += layerWarp(layer) * (0.58 * waveA + 0.42 * waveB);
    r += audioRadialOffset(layer, uBass, uMid, uTreble, uTransient);
    return vec2(cos(theta), sin(theta)) * r;
}

float du = 1.0 / uSamplesPerCurve;
vec2 c0 = centerline(aU, aCurveId, aLayer);
vec2 c1 = centerline(min(aU + du, 1.0), aCurveId, aLayer);
vec2 tangent = normalize(c1 - c0);
vec2 normal = vec2(-tangent.y, tangent.x);

// 相邻线形成稳定、均匀的并行线束
vec2 p = c0 + normal * aStrandOffset * uBundleSpacing;

// instanceId 解码后再进行扇区旋转与镜像
p.x *= mirrorSign;
p = rotate(float(rotationIndex) * TAU / float(uSectorCount)
         + uGlobalPhase) * p;

禁止使用相互独立的 gl.POINTS 来近似主体线条。WebGL2 对 lineWidth > 1 的支持不一致，因此推荐把相邻采样点展开为屏幕空间细带三角形：每个线段 2 个端点、左右各一个偏移顶点、组成 2 个三角形。这样可稳定得到 0.35–1.2 px 的抗锯齿线宽。若先做低成本验证，可用 1 px LINE_STRIP，但正式版本应使用 ribbon mesh。

建议 K 常态为 6 或 8。若需要在 6 / 8 / 10 之间切换，不能直接跳变；应同时绘制旧结构与新结构，并在 1.5–2.5 秒内交叉淡入淡出。对称阶数每 12–24 秒最多变化一次，不应跟随每个音频瞬态切换。

4.2 曲线束渲染分层

按以下顺序渲染，避免再次变成点云：

**主轮廓线：**占总亮度约 55%–65%，连续实线或高密度短划线，透明度 0.18–0.42。

**次级平行线：**占总亮度约 25%–35%，线宽更细、亮度更低，用于形成参考图的等高线／线束厚度。

**沿线微粒：**占总亮度约 5%–15%，粒子位置必须由 C(fract(u + speed * time)) 求得，沿已有曲线移动。

**交点高亮：**只在不同线束接近或音频瞬态发生时短暂出现，不可扩散为大片 Bloom。

允许部分曲线使用 dashMask = smoothstep(...) 做高密度点划纹理，但每段亮线的长度应明显大于线间空隙，使人眼首先读到“连续轨迹”，而不是离散圆点。

4.3 连续形变控制器

新增独立 MorphController，只产生低频、连续、确定性的参数，不在渲染帧中使用 Math.random()：

interface MorphState {
  sectorCount: 6 | 8 | 10;
  globalPhase: number;      // 极慢，约 0.015–0.05 rad/s
  layerPhase: Float32Array; // 各层可正反向漂移
  foldAmount: number;       // 0.15–0.85
  lobeDepth: number;        // 0.04–0.22
  outerReach: number;       // 0.45–0.82
  flowSpeed: number;        // 0.08–0.32
  topologyMix: number;      // 两组轨迹之间平滑插值
}

参数目标值每 4–9 秒缓慢更新一次，使用 smoothstep、阻尼弹簧或指数插值在 2–6 秒内过渡。不得直接切换半径、波形阶数或层级显隐。

推荐轨迹混合：

vec2 curveA = polarRibbon(...);   // 弧带/回环
vec2 curveB = radialTendril(...); // 放射触须
vec2 base = mix(curveA, curveB, smoothstep(0.0, 1.0, uTopologyMix));

4.4 拖尾与后处理

抗锯齿：优先使用屏幕空间 ribbon + smoothstep 边缘 alpha；线宽不得依赖世界坐标缩放

拖尾：从旧版 0.88–0.94 降至 0.68–0.82，只保留短暂运动余迹。图二、图三的层次主要由同帧内多条平行线形成，不应依靠长时间累积

Bloom：强度建议 0.05–0.16、阈值 0.72–0.88；仅增强交点和中心高光，不能模糊普通线条

主线颜色以银白／浅灰紫为主，普通线 alpha 保持较低；重叠处通过加法混合自然变亮

Fallback 档关闭 FBO，使用 1 px LINE_STRIP 直出；仅沿线微粒保留加法点精灵

4.5 必须避免的错误效果

使用大量独立圆形 point sprite 承担主体轮廓

通过增加粒子数量、点尺寸或 Bloom 来假装“线性感”

每个点独立采样噪声，造成椒盐抖动和块状聚集

线束只有 1 条粗线，没有平行等高线层次

主要轮廓依赖长拖尾形成，一暂停就消失

只有一个中心空洞的粗椭圆环

所有粒子同半径、同速度、同方向旋转

高频噪声导致边缘毛刺持续跳动

每个扇区随机不同，破坏万花筒秩序

Bloom 过强使内环糊成白色实心圆

外层触须恒定存在且长度一致，形成静态太阳图标

对称阶数瞬间跳变，造成画面闪断

仅用 sin(angle * N) 改圆周半径，最终退化为规则花朵

13. 给 AI 编程工具的任务指令（V1.4 线性视觉改造）

请在不破坏现有录音、音频分析、状态机、下载、Dock UI 和 V1.3 万花筒对称拓扑的前提下，重构 particle-field 的主要绘制原语。当前结果仍是一块块的颗粒云，原因是使用了大量互不连接的圆形 point sprite，并依靠点尺寸、随机散布、拖尾和 Bloom 填充轮廓。目标效果应接近参考图二、图三：由大量纤细、连续、间距稳定的参数曲线组成多层万花筒；不同帧之间轮廓持续顺滑形变，但每一帧本身都具有清楚的线性结构。

核心约束：

同一 MediaStream 用于分析与录音；启动后必须 resume AudioContext；采集约束失败要回退。

提取 volume、instantVolume、dbfs、hasSignal、bass、mid、treble、transient、centroid。

保留 MorphController，但所有形变必须作用于连续曲线 C(u)；禁止对每个顶点或粒子独立加白噪声。

seed 只在基础半扇区生成连续曲线，再通过 K 次旋转与镜像复制；默认 K=6，可在 6/8/10 间低频切换。

每条曲线沿 u 连续采样，并沿曲线法线生成 8–28 条平行线；正式版使用屏幕空间 ribbon triangles，不能继续用独立 gl.POINTS 作为主体。

同时渲染 Core、Inner、Mid、Outer 四层，各层半径、曲率、线束密度、相位和速度不同；外层包含可伸缩的连续触须／弧带。

主体亮度 85%–95% 来自线束；只保留 5%–15% 的辅助微粒，且必须沿 C(u) 切线方向移动。

音频只调制约 25%–40% 的运动幅度；自主生成运动占 60%–75%，确保无声时不静止。

对称阶数变化必须双图层交叉淡变 1.5–2.5 秒；禁止直接改 K 导致跳帧。

将拖尾降至 0.68–0.82，Bloom 强度降至 0.05–0.16；关闭后处理时仍必须看到完整线束拓扑。

禁止通过单纯增加粒子数、增大点尺寸、增强 Bloom 或延长拖尾来模拟线条。

保留底栏自适应、麦克风检测和 High→Medium→Low→Fallback 降档逻辑。

先完成一组不带音频的静态曲线束，再接入 MorphController，最后接入音频；用静音、持续元音、说话、拍手验证 A-13 至 A-22。

建议实施顺序：

先关闭旧粒子层、拖尾和 Bloom，只用一条 1 px LINE_STRIP 验证 C(u) 连续且形状正确。

沿法线复制为 8–28 条平行线，确认线间距稳定、无块状聚集。

将 LINE_STRIP 升级为带抗锯齿的屏幕空间 ribbon mesh。

把线束接入 rotationIndex + mirrorSign 与 Core/Inner/Mid/Outer 四层。

接入 MorphController，确认所有相邻顶点随时间连续移动。

加回少量沿线微粒，再以很低强度恢复拖尾和 Bloom。

最后微调音频映射，不先改录音和 UI 功能。
